小说阅读工具 2

发表于 2017-04-06

除了完成小说获取展示的基本功能。继续加了一些小功能：

  1. 搜索页面 - 列表页面 - 章节页面之间的跳转；
  2. 文字大小的调整
  3. D页面背景色的指定更改

界面通过bootstrap实现，动效通过jquery实现。由外部的CDN引入。

参考网址：

  1. bootstrap
  2. jquery

问题在于我使用chrome的开发者工具可以看到加载页面的速度有2-5秒，加载的文件大小有2-3M。两个数据我都很差，起码速度是能直接感受到，而文件大小实在太费流量了。『不知道对不对，对于速度，我觉得虽然是CDN，但是万一线路就是不好呢。。』引入服务器本地的相关文件。『当时操作时没有实时记录，所以具体数据不太清楚了。』总之，情况都不好。现在想到的方法有缓存以及数据压缩传输。当时想到获取小说时，源网站使用了压缩，所以google了一下gzip压缩。在nginx上配置好了之后，通过谷歌开发者可以看到两个数据都有了明显的降低。加载速度为1-2.5秒。加载文件的大小为100KB左右。

附上Nginx关于gzip模块的配置文档，翻译自http://nginx.org/en/docs/http/ngx_http_gzip_module.html

Module ngx_ http _ gzip_module

ngx_http _gzip _module模块是一个使用“gzip”方法压缩响应的过滤器。 这通常会将传输数据的大小减少一半或更多。

1. 配置示例

gzip on;

gzip_ min_ length 1000;

gzip_ proxied expired no-cache no-store private auth;

gzip_types text/plain application/xml;

$gzip_ratio变量可用于记录所获得的压缩比。

2. 指令

语法： gzip on | off;

默认： gzip off;

上下文：http, server, location, if in location

↑-启用或禁用对响应数据的压缩。

语法： gzip_buffers number size;

默认： gzip_buffers 32 4k | 16 8k;

上下文：http, server, location

↑-设置压缩响应数据的数量和大小。默认情况下，缓冲区大小等同与一个内存页。而大小为4k还是8k则取决于平台。

  • 0.7.28版本之前，默认设置为4个4k或者8k缓冲区。

语法： gzip_ comp_ level level;

默认： gzip_comp _level 1;

上下文： http, server, location

↑-设置gzip压缩响应的等级。等级范围为1-9。

语法： gzip_disable regex …;

默认：——

上下文：http, server, location

  • 该指令在0.6.23版本中出现。

↑-禁止对被正则表达式匹配到的“User-Agent”头字段的请求的响应进行压缩。（即可设置白名单，对此名单内的响应不压缩。）

特殊掩码“msie6”（0.7.12）对应于正则表达式“MSIE [4-6] \。”，但工作速度更快。 从版本0.8.11开始，“MSIE 6.0; … SV1”从此掩码中去除。（这句翻译的很奇怪。不好理解。）

语法： gzip_ min_ length length;

默认： gzip_ min_ length 20;

上下文：http, server, location

↑-设置压缩数据的最小数。（即小于这个数，则不进行压缩了。）该长度仅根据“Content-Length”响应头字段确定。

语法： **gzip_ http_ version 1.0 | 1.1;

默认： gzip_ http_ version 1.1;

上下文：http, server, location

↑-设置压缩响应所需的请求的最小HTTP版本。

语法： gzip_ proxied off | expired | no-cache | no-store | private | no_ last_ modified | no_etag | auth | any…;

默认： gzip_proxied off;

上下文：http, server, location

↑-根据请求和响应启用或禁用代理请求的响应的gzip压缩。 具体是代理请求由“Via”请求头字段的存在来确定。指令接受多重参数：

  • off : 禁止所有的代理请求的压缩，忽略其他参数
  • expired : 如果响应头包括具有控制缓存时间的“Expires”字段与其值，则启用压缩。
  • no-cache : 如果响应头包含“Cache-Control:no-cache”,开启压缩。
  • no-store : 如果响应头包含“Cache-Control:no-store”,开启压缩。
  • private : 如果响应头包含“Cache-Control:private”,开启压缩。
  • no_ last_ modified : 如果响应头没有包含“Last-Modified”,开启压缩。
  • no_etag : 如果响应头没有包含“ETag”,开启压缩。
  • auth : 如果响应头包含“Authorization”,开启压缩。
  • any : 为所有的代理请求都开启压缩。

语法： gzip_types mime-type…;

默认： gzip_type text/html;

上下文：http, server, location

↑-除了“text/html”，也可以对其他指定的MIME类型的响应进行压缩。特殊字符“*”可匹配所有的Mime类型（0.8.29）。“text/html”类型的响应则总会被压缩。

语法： gzip_vary on | off;
默认： gzip_vary off;
上下文：http, server, location

↑-当指令gzip,gzip_static,或者gunzip开启，允许或者禁止插入“Vary:Accept-Encoding”响应头域。

3.嵌入变量

  • $gzip_ratio：实现的压缩比，即原始和压缩响应大小之间的比率。
