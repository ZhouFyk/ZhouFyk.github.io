微信 token 验证失败记录

发表于 2017-06-21

之前尝试过了解微信开发，不过因为帐号权限的问题，没有进一步的了解。所以只有一个简单的自动回复：

xxx 是我室友的名字。
最近想在微信上弄一个东西，随手发了信息，却发现自动回复：

该公众号暂时无法提供服务，请稍后再试。

我都没有动过，所以也不知道是怎么回事。进后台看了一下配置，发现虽然我对微信相关的部分没有动过，但是我后来弄了域名，所以之前微信使用的 IP 就不对了，于是更改了一下。然后报错：

对比 微信公众平台技术文档 中的流程图：

服务器上的验证代码的逻辑应该是没问题的。查看错误日志，看到了发来的请求中的各个参数，直接拿参数来验证发现确实是没有问题的。代码没问题，以及微信的服务器发来的请求也没问题的话，再看看错误日志中的其他信息，发现如下错误：

看起来似乎是 fastcgi 相关的问题。google 之后的解释有：

大概就是要找的文件没找着。想半天，再把错误信息仔细看看，瞎了我的眼。请求的路径有问题啊。(┬＿┬)。怎么多了个目录。再仔细看配置文件是否配置错了，也没找出来错误。还是看错误信息，(┬＿┬)，瞎了我的眼，最后的错误信息是之前的，现在的都没问题了。不过 token 验证还是错误。因为官方文档是用 python 写的，但是之前可能是 php 写的，因为有搜到微信接口配置token验证失败。，其中楼主贴的官方文档代码是 php 的。对比了一下，发现该代码中验证了之后直接 echo $echostr。而我因为之前验证过了，所以没有这部分的代码，然后我刚刚自己弄的时候，是 return $echostr。可能是这个有问题。这样修改了之后，验证成功了。不过现在的官方文档中的 python 代码也是 return echostr，不知道是不是语言语法的问题。

总结：

本次的问题相关有 nginx 的配置文件，以及日志查询，和微信方面的文档。最后解决问题的办法，可能是官方定义的验证方式就是要直接输出 echostr 吧。
