CI - Core - Common 记录

发表于 2017-07-01

github - CodeIgniter/system/core/Common.php

PHP 版本确定

is_php($version)：
检测当前 PHP 的版本是否高于指定版本。

  • version_compare(PHP_VERSION, $version, '>=');

检测文件/目录是否可写

is_really_writable($file)：

确定系统，通过目录分隔符判断 if (DIRECTORY_SEPARATOR === '/')，如果是 UNix 类的系统，直接使用 is_writable($file)。

如果参数为目录，尝试在该目录内创建新文件，创建失败即不可写，创建成功则删除创建的文件，返回 true。

如果不是目录，通过操作符 OR 将「判断是否是文件」以及「在该文件尝试写入」的操作连接起来：! is_file($file) OR ($fp = @fopen($file, 'ab')) === FALSE，返回判断结果。

在该过程中对文件或目录进行操作，要及时关闭操作句柄。

加载指定类

&load_class($class, $directory = 'libraries', $param = NULL)：

设置一个静态变量数组，将实例化了的类对象存入其中，如果有再次调用此函数，则直接返回已经存在的对象。所以本函数名前加了 &，按引用返回变量 - 引用返回：

class foo {
public $value = 42;

public function &getValue() {
return $this->value;
}
}

$obj = new foo;
$myValue = &$obj->getValue(); // $myValue is a reference to $obj->value, which is 42.
$obj->value = 2;
echo $myValue; // prints the new value of $obj->value, i.e. 2.

和参数传递不同，这里必须在两个地方都用 & 符号——指出返回的是一个引用，而不是通常的一个拷贝，同样也指出 $myValue 是作为引用的绑定，而不是通常的赋值。。

与引用传参不同，函数或者方法中引用传递参数，只需要在定义函数或者方法时，在变量前添加 & 符号，实际使用时无需再使用 - 引用传递：

注意在函数调用时没有引用符号——只有函数定义中有。

  • 函数-返回值
  • 引用的解释。

这个函数的终极目的就是返回一个实例化的类对象。前面已经说了如果已经有了对象就直接返回。那么没有对象，肯定是要 new 一个的。将这个对象保存在静态数组中，然后返回，这样这次的对象获取了，下次还要用到就可以直接返回。

那么 new 一个对象的前提是一定要有这个类的存在，该类已经定义了。那么我们要确认一个类是否已经定义，我们要先检测这个类的文件存在不存在，存在的话，再把这个类文件通过 require_once 包含进来，不存在自然报错。

那么在检测包含类文件的这个过程中，就是 CI 框架中的类，项目中创建新类以及扩展类的加载逻辑。它得先检测这个类文件存在不存在，存在的话呢，要检测 CI_classname 这个类是否已经定义了。这个类呢，是替换原核心类的 - 替换核心类，就是用户觉得啊，这个原本的核心类用起来不怎么好使，就自己再写一个，那要用的自然是用户自己写的这个类，所以在这段逻辑里，类文件存在之后，新核心类已经定义了，就没啥事了，如果没有用户创建的新核心类，就把框架的核心类包含进来。这个过程呢，要从框架的类库里和应用目录的类库里依次过一遍，找着了就跳出循环到下一阶段，找不着呢，也到了下一阶段了。

这个下一阶段呢，就是找扩展的核心类 - 扩展核心类，这个扩展的核心类，就是用户觉得原来的类咋不是那么好使，缺点啥，就自己从原来的基础上扩展了一下。自然，用户要使用的肯定是自己扩展过的类。同样，检测扩展类的文件是否存在，存在的话，看看这个类是否已经包含了，没有就包含一下。

到这里，如果以上流程走下来，都没有成功，就是没有这个类，那就报个错：嘿，这里有问题。

最后实例化对象就大功告成了。

这个函数的注释是这样的：

This function acts as a singleton. If the requested class does not exist it is instantiated and set to a static variable. If it has previously been instantiated the variable is returned.

所以这个函数的主要思想是扮演一个单例的角色，减少资源，一个对象走遍整个应用流程。

记录已经实例化的类

&is_loaded($class = '')：

这个函数就是记录已经实例化了的类。没别的作用了。而且只被上面 load_class() 调用。

加载配置

&get_config(Array $replace = array())：

同样是静态数组，用来存储加载了的配置参数。看到静态变量，我们大概就能知道它的用法了。这个数组里面有配置呢，就对这些配置操作，没有呢，就要加载进来保存在这。

当然，一个应用刚开始肯定都是空的嘛，那这个函数被调用的时候就是要从配置文件中加载配置。首先，看看基础配置的文件存在不存在，存在就包含进来。再查看是否存在特定环境下的配置文件，有的话也加载进来。

然后又是错误处理，如果这些文件都不存在，就报个错，处理一下。过了这个错误，那就是配置文件是存在的。然后呢，不能直接使用，还要检测。检测啥，检测配置文件里有没有我们需要的配置数组，万一是个空的，那就没办法用了。所以这里又是个错误处理，看看配置数组存在不存在，是不是个数组类型。过了这个错误，配置就可以用了。

不过呢，这样配置不就是死的了吗，只能写在配置文件里，那就不太好修改了。所以这里有个对配置文件的处理，这个函数的参数就是要操作的新参数，原配置数组里有的，就给改了，没有的，就赋值进去，这样就达到在应用运行过程中能够修改配置的目的了。这样做完了之后，返回配置数组。

获取指定配置参数

config_item($item)：

实现方式和 &get_config() 差不多，对于一个获取指定数据的函数来说，可以很直观的想出它的实现逻辑：获取所有的参数，如果存在这个参数就返回，否则返回 Null。

这里所有的配置信息通过 &get_config() 来获取。这个函数中，有一个唯一的注释：

references cannot be directly assigned to static variables, so we use an array.

因为这个函数中也设置了一个静态变量，为什么设置这个，原理同之前讲过的静态变量一致。获取配置信息在这个函数中的代码实现是这样的：$_config[0] =& get_config();，直觉上这里会觉得加上一个索引值很奇怪。所以这里的注释是对这个的解释。更具体的，经由 google：

  • PHP general question

大概就是如果直接赋值，这个值并不会被保存如静态变量中，下一次再次使用会发现这个静态变量里面是空的。不过当做数组来使用，就没问题了。

这个问题里面的答案，有从手册引用的部分，这部分引用的内容较好的解释了这个问题。然而我现在查手册，发现并不完全一致，关键部分有差异。

总之这是个知识点，要记住。

&get_mimes()：

基本同以上的函数，存在则返回，不存在则包含，包含过程中，检测文件，包含文件，不存在文件设为空一条龙。

是否使用 https

is_https()：

通过预定义变量 $SERVER 的三个值来判断：

  • $_SERVER['HTTPS']

    如果脚本是通过 HTTPS 协议被访问，则被设为一个非空的值。

    Note: 注意当使用 IIS 上的 ISAPI 方式时，如果不是通过 HTTPS 协议被访问，这个值将为 off。

  • $_SERVER['HTTP_X_FORWARDED_PROTO']

  • $_SERVER['HTTP_FRONT_END_HTTPS']

但是这个三个值优先级依次从高到低。如果前一个变量存在，设置正确就会直接返回 true，只有在前一个变量不正确时才会判断下一个变量。

输出错误信息

show_error($message, $status_code = 500, $heading = 'An Error Was Encountered')：

对传入的状态码进行简单的处理，然后通过 & load_class() 实例化 Exceptions 类，调用该类的 show_error() 输出错误信息。

对状态码处理是防止无效的状态码，比如状态码没有小于 100 的，所以如果传入的状态码小于 100，就把状态码置为 500。同时设定一个结束状态码：$exit_status = $status_code + 9; // 9 is EXIT__AUTO_MIN，注释可知，这是 CI 框架内部定义的一个常量。

输出404

show_404($page = '', $log_error = TRUE)：

从代码来看的话，这个函数是上面那个函数的特定情况，这个函数只处理 404 的错误。同样是通过 & load_class() 实例化 Exceptions 类，调用该类的 show_404() 输出错误信息。

记录信息

log_message($level, $message)：

同之前的函数，一个静态变量，如果该变量为空，就作为数组保存实例化了的 Log 类对象，然后调用该对象的 write_log() 方法记录相关信息。

设置 HTTP 状态头信息

set_status_header($code = 200, $text = '')：

先检测 PHP 的模式，如果是命令行，就不需要 HTTP 信息了。再检测状态码是否存在，是否是数字，不是的话报错。如果状态信息为空，就采用默认设定的文本信息。如果默认信息没有的话，那就得报个错了。

如果信息都没问题的话，接下来就需要发送出去了。关于这个状态信息，PHP 的模式里有两个模式会用到，格式不一样，所以要判断一下模式，组成合适的格式发送出去。

cgi 模式直接摘录代码：

if (strpos(PHP_SAPI, ‘cgi’) === 0)
{
header(‘Status: ‘.$code.’ ‘.$text, TRUE);
return;
}

判断条件里的比较运算符为 ===，而不是 ==，是因为函数 strpos() 的作用是 「查找字符串首次出现的位置」，它的返回值是：

返回 needle 存在于 haystack 字符串起始的位置(独立于 offset)。同时注意字符串位置是从0开始，而不是从1开始的。

如果没找到 needle，将返回 FALSE。

所以如果找不到，此时使用比较运算符 == 就会出错，因为 false == 0 是 true。所以使用 === 进行更严格的类型判断，这样就没有问题了。

然后就是格式的问题，我还是第一次知道 CGI 这个头信息状态行的样子 - Status:200 ok。平时在网页上看到最多的是 http1.0 200 OK 这样的形式。

过滤 html

html_escape($var, $double_encode = TRUE)：

传入的参数为空，直接返回该参数。传入的参数类型为数组，则通过 foreach，递归调用本函数，最后返回过滤后的值。如果是单个字符串直接使用 htmlspecialchars()，来处理，返回处理结果。

字符串属性转换

_stringify_attributes($attributes, $js = FALSE)：

对参数 $attributes 进行检测，为空直接返回空。是字符串的话，直接返回该参数。如果是其他类型，则强制转换为数组类型。然后通过 foreach，以及传入的第二个参数的值来拼接属性字符串，最后返回。

检测函数是否可用

function_usable($function_name)：

设置一个静态变量数组用来保存函数黑名单。如果传入的方法存在，然后黑名单不存在，就看看配置文件里有没有黑名单，没有的话，黑名单就赋个空数组，有就直接赋值黑名单数组。然后看看黑名单里面有没有这个函数，返回相应的布尔值。当然，如果方法不存在的话，就直接返回 false 了。

总结

经过以上的说明，可以看到大部分的函数有一些类似。比如都使用了单例的思想，将一些对象或属性保存在静态变量中，除了第一次的实例化之外，之后都可以直接返回相应数据，以提高资源使用率。

还可以学习到一些对我来说更好的代码实现方式，不管是逻辑还是写法。以及学习到 PHP 的一些以前没有注意到的小知识，像位运算符和静态变量的引用赋值等。
