ci - core - hooks

发表于 2017-07-10

CI_Hooks

该类的作用是提供一种机制在不对框架核心代码进行修改的情况下定制框架流程。

属性

  1. $enabled：钩子是否开启的标识符。
  2. $hooks：配置目录 config 中钩子配置中的所有钩子的列表。
  3. $_objects：使用了钩子方法的类对象的列表。
  4. $_in_progress：钩子运行标识符。用来标记钩子运行状态，防止陷入无限循环当中。

构造方法

__construct()：

加载配置，看钩子是否被启用，没有启用直接返回。然后加载所有存在的钩子的配置文件，看看这些文件中是否有实际存在的钩子的配置信息，没有也就直接返回了。如果存在的话，就保存到属性里，同时打开钩子开启的标识符。

调用钩子

call_hook($which = '')：

这个方法在主流程文件 Codeigniter.php 中被调用，用来调用指定的钩子。

首先是错误检测，看钩子是否启用，再判断指定的钩子存在与否。不正确则直接返回。

如果调用的指定的钩子配置信息是一个数组，并且其中没有函数的配置信息，那就通过 foreach 获取该数组的值，依次通过方法 _run_hook() 来运行。如果不是数组，就直接使用方法 _run_hook() 操作该值。一切结束返回 true。

运行指定的钩子

_run_hook($data)：

首先检测传入的参数是否可调用，由注释可知 Closures/lambda functions 以及 array($object, 'method') 形式的参数能够被调用，所以这里检测参数是否是属于这两类。如果可以被调用，则区分这两类，分别调用方法。如果不可调用，而且参数又不是数组，则返回 false。

而通过以上步骤，到达这里，就说明参数不是方法不能被调用，并且是数组。

然后检测当前钩子对象是否在运行，目的是防止出现调用已经调用了的钩子而陷入死循环。

设置文件路径。看参数中是否有路径相关的键值对，没有返回 false。存在则处理成一个完整的路径。接下来就是检测这个路径了，如果这个路径是无效的，就返回 false。

通过以上，到达这里，就说明文件存在。此时就确定参数中是否存在类、方法和参数了。设置好这三个值，然后逐一判断。首先是方法，方法不存在的话，就没办法调用了，直接返回 false。方法存在，则表示钩子可以运行，就将标识符设置成在运行。然后检测类是否存在，如果存在，看看该类的对象是否已经存在于列表中，如果存在，就检测该类对象的方法是否存在，存在调用方法，不存在钩子运行标识符设置成 false。如果类不存在与配置文件中，就搜搜看，找不着类，或者找到的类中没有指定的方法，就将钩子运行标识符置为 false，返回。如果一切没问题，自然就调用了。

如果类一开始就不存在的话，就找找方法，找不着也就 false 了。找的着就调用了。

那么到这里一切的操作都完成了。钩子都调用成功了。就把标识符关了，返回 true，表示调用钩子没问题。

总结
