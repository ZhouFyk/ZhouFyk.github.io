Lang.php：语言类

属性

  1. $language：加载了的语言对应翻译列表。
  2. is_loaded：已经加载的语言文件。

构造方法

__construct()：

记录日志信息，语言类已经实例化。

加载语言文件

load($langfile, $idiom = '', $return = FALSE, $add_suffix = TRUE, $alt_path = '')：

最重要的参数是 $langfile 对吧，毕竟没有设置默认值。那么首先要对其进行检测，检测是不是数组，如果是数组就要通过 foreach 循环，内部调用本方法。

经过 foreach 的处理之后，语言包文件名都是标量了。然后要对这个文件名去扩展名处理。为什么要去扩展名呢，因为方法的一个参数是后缀的标识符啊，如果为 true，就要添加后缀。代码中的处理方法就是，去掉扩展名，加上后缀，再把扩展名添上去。而从添加后缀的代码看来，这个后缀是 _lang，代码是 preg_replace('/_lang$/', '', $langfile).'_lang'，这么写的原因应该是无法确定输入的语言包文件名是否有后缀，那就干脆重新设置一遍。这么处理了之后，再重新添上文件扩展名。

然后就是处理第二个参数语言名称 $idiom，如果语言名称不符合要求，比如为空或者字符非法，就从配置中获取语言名。如果配置中有就使用配置中的语言名称，否则设为英语。

确认语言包的必要因素都知道了，就看看这个语言包是否已经正确加载过了，正确加载过了，并且根据参数不需要返回加载列表，就直接返回。

当语言包未正确加载，就根据框架的设定，看看默认的基础语言包目录中是否存在该语言包，有则加载。

到最后一个参数了。最后一个参数是替代路径。如果替代路径存在，则说明用户是指定了这个路径的语言包，我们同样查看该路径中是否存在该语言包，存在则加载。

如果没有替代路径，那就从所有的包路径中检测是否存在指定语言包，存在则加载。

如果经过以上三次检测加载语言包的流程都没有成功加载，那就报个错吧。

如果加载了语言包呢，也就是文件存在并成功加载，并不是大功告成了。和其他源代码中加载文件部分一样，即使成功加载了文件，也得检测文件中是否存在我们需要的内容，否则返回 true，实际上啥也没有，这算成功吗？所以检测语言相关变量 $lang 是否存在，是否是数组形式，来判断加载的文件是否有数据，没有数据就记录日志信息，根据 $return 来决定直接返回空数组，或者直接返回。当然如果有数据，对于返回来说也是一样的，要么返回数据，要么记录下加载了的语言包信息再返回。

返回指定的配置参数

line($line, $log_errors = TRUE)：

看属性 $language 数组中是否已经加载了，存在则返回值，不存在根据错误记录标识符来决定是否记录错误日志再返回 false。
