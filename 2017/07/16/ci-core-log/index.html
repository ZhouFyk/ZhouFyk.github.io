ci-core-log

发表于 2017-07-16

Log.php

属性

  1. $_log_path：存储日志文件所在路径。
  2. $_file_permissions：文件权限。
  3. $_threshold：日志等级。
  4. $_threshold_array：日志等级数组。
  5. $_date_fmt：日志文件时间戳格式。
  6. $_file_ext：文件扩展名。
  7. $_enabled：是否开启日志记录的标识符。
  8. $_levels：预定义日志记录等级。
  9. $func_overload：mbstring 扩展方法重载标识符。

构造方法

__construct()：

获取配置信息。确定 mbstring 扩展的方法重载标识符。设置日志路径属性。设置文件扩展名属性。检测日志文件路径是否正确，不正确即日志文件不存在则创建该目录。

如果日志路径不是目录，或者该目录不可写，则日志记录标识符设为 false。

以上属性的设置基本都是基于获取的配置文件信息来设置的。

记录日志

write_log($level, $msg)：

如果日志记录没有开启，返回 false。如果日志记录等级不符合要求，返回 false。设置当前时间格式的日志文件路径。检测该日志文件是否存在。如果不存在，就设置一个新文件的标识符，将扩展名设置为 php，在 $message 变量中添加验证信息：<?php defined('BASEPATH') OR exit('No direct script access allowed'); ?>\n\n。尝试创建写入该文件，失败返回 false。如果成功，锁定该文件。设置时间。在信息变量中添加日志行。

写入日志信息。摘录代码：

以前很少用过文件相关的函数。此处的写法对我来说也是很新鲜。$written 为写入字符数，$length 为日志信息总字符长度。循环条件是写入字符数小于总字符数，每次循环完，写入字符数增加 $result。循环内一行代码实现写入功能。使用 fwrite 往文件中写入字符串 self::substr($message, $written)，即信息中还未写完的字符串。$result 为该次写入了多少字符。

记录完日志，对日志文件解锁，关闭文件资源句柄。

如果当次记录日志是新的日志文件，则修改文件权限。

最后将最后一次循环记录日志字符串数目作为结果返回。

总体来说，CI 日志记录的方式还是十分简单的。核心方法就一个 write_log($level, $msg)。从中了解了常规日志的记录方式，和一些小技巧。
