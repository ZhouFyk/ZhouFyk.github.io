Mdn: http访问控制(cors)

跨域资源共享：cors (cross-origin resource sharing)

当一个资源从与该资源本身所在的服务器不同的域或端口请求一个资源时，资源会发起一个跨域 HTTP 请求。比如当前浏览的是 a 站点，但是请求的图片来自于 b 站点，那 a 站点就会发起一个跨域 http 请求。

CORS 需要客户端和服务器同时支持。

简单描述：使用 http 首部字段来进行访问权限的控制。首先是对安全没有太大影响的会通过，比如单纯的 get 方式，把这种称为简单请求。而一些有风险存在的情况，则会对其进行权限控制，比如 put, delete 等等，这就需要一个预先检查，看服务器允许不允许该种情况的发生，把这种称为预检请求。服务器允许之后，在服务器允许的情况下也可以做一些其他操作，比如设置 cookies 等，得到服务器的正面响应，然后客户端再发起实际的请求。

简单请求：符合以下所有条件的即是 简单请求

  • 请求方法为以下之一：
    • GET
    • HEAD
    • POST
  • Fetch 规定的对 cors 安全的首部集合，这些集合之外的首部都是不被允许的：
    • Accept
    • Accept-Language
    • Content-Language
    • Content-Type
    • DPR
    • Downlink
    • Save-Data
    • Viewport-Width
    • Width
  • Content-Type 为下列三值之一：
    • text/plain
    • Multipart/form-data
    • Application/x-www-form-urlencoded
  • 请求中的任意 XMLHttpRequestUpload 对象均没有注册任意事件监听器；XMLHttpRequestUpload 对象可以使用 XMLHttpRequest.upload 属性访问。
  • 请求中没有使用 ReadableStream 对象

预检请求：预检请求可以避免跨域请求对服务器的用户数据产生未预期的影响。当满足以下任意条件时，应该发送预检请求：

  • 使用了以下任一 HTTP 方法：
    • PUT
    • DELETE
    • CONNECT
    • OPTIONS
    • TRACE
    • PATCH
  • 人为设置了 对 cors 安全的首部字段集合 之外的其他首部字段。该集合为：
    • Accept
    • Accept-Language
    • Content-Language
    • Content-Type
    • DPR
    • Downlink
    • Save-Data
    • Viewport-Width
    • Width
  • Content-Type 的值不属于下列之一：
    • application/x-www-form-urlencoded
    • multipart/form-data
    • text/plain
  • 请求中的任意 XMLHttpRequestUpload 对象注册了任意多个事件监听器；
  • 请求中使用了 ReadableStream 对象

Mdn:服务器端的访问控制 - PHP

简单请求发起的情况：

  • 请求方式为 GET 或者 POST，如果是 POST，那么请求的 Content-Type 为以下之一：application/x-www-form-urlencoded, mumtipart/form-data, text/plain。
  • 请求中无自定义的头部（如 X-Modified）

以下情况，请求会返回相关响应信息

  • 如果资源是允许公开访问的(就像任何允许 GET 访问的 HTTP 资源),返回 Access-Control-Allow-Origin:* 头信息就足够了,除非是一些需要 Cookies 和 HTTP 身份验证信息的请求。
  • 如果资源访问被限制基于相同的域名,或者如果要访问的资源需要凭证(或设置凭证),那么就有必要对请求头信息中的 ORIGIN 进行过滤,或者至少响应请求的来源(例如 Access-Control-Allow-Origin:http://arunranga.com)。另外,将发送 Access-Control-Allow-Credentials:TRUE 头信息，这在后续部分将进行讨论。

预检请求发生情况：

  • 使用 GET 或 POST 以外的情况；使用 POST 发送 application/x-www-form-urlencoded, multipart/form-data, text/plain 之外的 Content-Type，如 application/xml
  • 发送自定义的头信息，如 x-pinghello

响应 preflight requests 请求的服务器资源会有这些动作：

  • 基于 ORIGIN 进行过滤
  • preflight请求的响应内容，包括必要的 Access-Control-Allow-Methods, Access-Control-Allow-Headers (保证系统正常运行)，如果需要凭据的话，也会包括 Access-Control-Allow-Credentials 头信息
  • 响应实际请求，包括处理 POST数据等。

凭证请求：带凭据的请求，将 Cookies 和 HTTP 认证信息一起发送出去的跨域请求，根据请求方式，可以是 简单请求 也可以是 预检请求。

发送 简单请求 时， Firefox 3.5 及以上会发送带 Cookies 信息的请求（如果 withCredentials 设为 true）。如果服务器响应客户端是可信任的，客户端接受并进行输出。在 预检请求 中，服务器可以针对 OPTIONS 请求，返回 Access-Control-Allow-Credentials: true 信息。
